│ ⚙️ CONFIG COMPARISON                                                      │
├───────────────────────┬──────────────┬──────────────┬─────────────────────┤
│ Parameter             │ ViT5-Base    │ ViT5-Large   │ Note               │
├───────────────────────┼──────────────┼──────────────┼─────────────────────┤
│ d_model (hidden)      │ 768          │ 1024         │ +33%               │

│ FFN dim              │ 3072         │ 2816         │ Slightly smaller  │
│ Vocab size           │ 36,096       │ 36,096       │ SentencePiece      │
│ Total params         │ ~220M        │ ~770M        │ 3.5x larger       │
│ Memory (FP16)        │ ~0.44GB      │ ~1.54GB      │ 3.5x              │
│ Inference speed      │ ~150 tok/s   │ ~50 tok/s    │ Slower but better │
└───────────────────────┴──────────────┴──────────────┴─────────────────────┘

┌──────────────────────────────────────────────────────────────────────────┐
│ 🚀 QUICK CODE                                                            │
├──────────────────────────────────────────────────────────────────────────┤
│ from transformers import AutoTokenizer, AutoModelForSeq2SeqLM           │
│ import torch                                                             │
│                                                                           │
│ tokenizer = AutoTokenizer.from_pretrained("VietAI/vit5-base")          │
│ model = AutoModelForSeq2SeqLM.from_pretrained("VietAI/vit5-base")      │
│ device = torch.device("cuda" if torch.cuda.is_available() else "cpu")  │
│ model.to(device)                                                         │
│                                                                           │
│ text = "tóm tắt: Đây là văn bản cần tóm tắt. </s>"                      │
│ inputs = tokenizer(text, return_tensors="pt").to(device)               │
│ outputs = model.generate(**inputs, max_length=50)                       │
│ result = tokenizer.decode(outputs[0], skip_special_tokens=True)        │
│ print(result)                                                            │
└──────────────────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────────────┐
│ ⚡ GENERATION PARAMETERS                                                 │
├──────────────────────────────────────────────────────────────────────────┤
│ Quality:   num_beams=4-8, length_penalty=2.0, early_stopping=True      │
│ Speed:     num_beams=1, do_sample=True                                  │
│ Diversity: temperature=0.8, top_k=50, top_p=0.95                       │
│ Length:    min_length=20, max_length=100-200                           │
└──────────────────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────────────┐
│ 🆚 ENCODER-DECODER vs DECODER-ONLY (ViT5 vs Qwen)                       │
├──────────────────────────────────────────────────────────────────────────┤
│ ViT5: Encoder-Decoder → Input → Encoder → Cross-Attn → Decoder → Output│
│      Best for: Translation, Summarization, QA                           │
│      Needs: Task prefix ("tóm tắt:", "dịch:")                          │
│                                                                           │
│ Qwen: Decoder-only → Causal LM → Token by token                         │
│      Best for: Chat, Creative writing, Code                              │
│      Needs: Chat template (System/User/Assistant)                       │
└──────────────────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────────────┐
│ 🔧 TROUBLESHOOTING QUICK FIXES                                           │
├──────────────────────────────────────────────────────────────────────────┤
│ Problem: "CUDA out of memory"                                            │
│ Fix: Use FP16: torch_dtype=torch.float16                                 │
│      Reduce batch_size or beams                                          │
│                                                                           │
│ Problem: Output gibberish                                                 │
│ Fix: Add task prefix: "tóm tắt: " + text + " </s>"                      │
│      Set proper max_length                                               │
│                                                                           │
│ Problem: Output too short                                                │
│ Fix: length_penalty=2.0, min_length=30                                  │
│                                                                           │
│ Problem: Model not loading                                                │
│ Fix: pip install --upgrade transformers sentencepiece                    │
│      Clear cache: rm -rf ~/.cache/huggingface                           │
└──────────────────────────────────────────────────────────────────────────┘

├──────────────────────────────────────────────────────────────────────────┤
│ Use vit5-base if:     • Quick prototyping                                │
│                       • Limited GPU memory                                │
│                       • Need fast inference                               │
│                       • General-purpose tasks                             │
│                                                                           │
│ Use vit5-large if:    • Maximum quality needed                           │

│                       • Want best-in-class performance                    │

┌──────────────────────────────────────────────────────────────────────────┐
│ 💡 KEY INSIGHTS                                                           │
├──────────────────────────────────────────────────────────────────────────┤
│ 1. ViT5 is encoder-decoder (like T5), NOT decoder-only (like GPT)       │
│ 2. Always use task-specific prefix ("tóm tắt:", "dịch:")                │
│ 3. Base model sufficient for most Vietnamese NLP tasks                  │
│ 4. Large model gives +1-2 ROUGE points but 3.5x bigger                  │
│ 5. Finetuned variants available for summarization & translation         │
│ 6. Works great with diacritics (dấu) out-of-the-box                     │
│ 7. Can run on CPU (slow) but recommend GPU for production               │
└──────────────────────────────────────────────────────────────────────────┘
